## Adaptive Notes

- Docker build for Part 05 exposed a dependency-resolution problem with `pydantic==2.9.0` / `pydantic-core==2.23.2` from the package index. The Part 05 production requirements were simplified to let FastAPI resolve Pydantic and to use plain `uvicorn==0.30.0` because the scaling demo does not require `uvicorn[standard]`.
- Direct smoke testing Part 05 exposed an off-by-one bug in `turn`; it returned `2, 3` for the first two messages. `05-scaling-reliability/production/app.py` now returns `1, 2`.
- A later Part 05 Docker build retry was blocked by Docker Hub DNS/connectivity while resolving `python:3.11-slim`, before application layers ran.
